- Big picture: Once the HJB/HJ link is visible, it lines up continuous‑time RL with physics and opens a control‑theoretic lens on modern generative modeling—interpreting diffusion model training as a stochastic optimal control problem, adjacent to optimal transport.

### Mr. Chatterbox is a (weak) Victorian-era ethically trained model

Mr. Chatterbox: a Victorian-only LLM you can run locally (and it’s delightfully bad)

Trip Venturella released “Mr. Chatterbox,” a 340M‑parameter language model trained entirely from scratch on 28,035 British Library books published 1837–1899 (about 2.93B tokens). Simon Willison took it for a spin, wired it into his LLM CLI via a new plugin, and reported back: the vibe is charmingly Victorian, but the utility is closer to a Markov chain than a modern chatbot.

Highlights
- Corpus and ethics: pretraining uses only out‑of‑copyright texts from the Victorian era; weights are about 2.05GB and run locally. Training used Karpathy’s nanochat.
- How it performs: fun prose style, poor task performance. Willison notes it rarely answers questions usefully.
- Why it struggles: Chinchilla’s 20× rule suggests ~6.8–7B tokens for a 340M model; this has ~2.9B. Willison suspects 4× more data might be needed to feel “useful.” For comparison, Qwen 3.5 only gets interesting at 2B+ params.
- Try it: there’s a Hugging Face Space demo, and Willison published an llm-mrchatterbox plugin so you can chat with it via his LLM CLI.
- Update on “pure” training: initial attempts to make it conversational with Wilde/Shaw plays or mined dialogue failed. The working approach used synthetic dialogue pairs generated by Claude Haiku and GPT‑4o‑mini for SFT—effective, but it weakens the “no post‑1899 inputs” claim.

Why it matters
- A concrete, reproducible test of whether a capable LLM can be built from public‑domain data only—an ethical and legal ideal many in the community want.
- Early result: style transfer is easy; competence is hard without lots more high‑quality tokens (and likely bigger models).
- The plugin story is a nice demo of using coding copilots (Claude Code) to scaffold real tooling around open‑weight models.

Bottom line: Mr. Chatterbox is more historical curiosity than assistant today—but it’s a promising, transparent step toward “ethically trained,” local LLMs.
