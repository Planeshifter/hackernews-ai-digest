Unlike always-on tools like caffeinate or Amphetamine, this open-source macOS menu bar app blocks sleep—including lid-closed clamshell sleep—strictly for the duration of active agent tasks, then gets out of the way so normal sleep resumes.

Why it’s interesting
- Agent-aware, not blanket awake. Hooks into popular AI coding agents (Claude Code, Codex, Cursor, Gemini CLI, Aider, Hermes, OpenCode, Cline, Pi) so the system stays up only while a turn/job is in progress.

- Thoughtful safety. Thermal cutout force-releases holds if skin/CPU temps spike with the lid closed, and idle/process-dead sessions auto-release to avoid runaway wakes.
- Low-friction integration. Sub-50ms CLI acquire/release calls from agent hooks; reference-counted assertions handle overlaps; optional process sniffing if you don’t want to install hooks.
- Nice UX touches. A lid-close chime confirms a hold when the screen is off; reopening shows a summary of what ran, peak temps, and whether a thermal cutoff fired.
- Clean install/uninstall. One-click installer wires hooks; uninstaller removes them.

Details
- macOS 26.4+ (author’s test target); Xcode 26+ to build; admin rights requested once to register the privileged helper.
- Signed/notarized DMG available; source in Swift 6 with strict concurrency; shared logic ships as a Swift package.
- Architecture and security model documented; helper + daemon registered via SMAppService; hooks call: adrafinil acquire/release <session-key>.
