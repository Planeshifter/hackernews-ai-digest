### Running local models is good now

Local LLMs hit “good enough”: Gemma 4 enables agentic coding on an M2 Mac

- The gist: After years of tinkering, the author says local models finally feel viable for day‑to‑day dev work. On a 2022 M2 Mac (64 GB RAM), Gemma 4 models are enabling agentic coding loops at roughly 75% of frontier models’ speed/accuracy—good enough that they’re no longer constantly cross‑checking with cloud APIs.

- What they ran:
  - Models: Mistral 7B, Gemma 3/4 (notably gemma-4-26b-a4b and gemma-4-12b-qat), OpenAI GPT‑OSS/OSS‑20B, Qwen 3 MoE and Qwen 2.5 Coder variants.
  - Runtimes: llama.cpp (and llama-cpp-python), Ollama, LM Studio, raw llamafiles, Open WebUI.

- What got done locally:
  - Agentic coding: refactored a Python notebook into a multi‑module repo; tightened type hints; generated unit tests; proofread posts; bootstrapped a simple two‑tower recommender repo—all offline.
  - Practical note: KV cache can balloon; they’ve seen RAM use climb toward the full 64 GB during longer sessions.

- Why this feels new:
  - “Good enough” inflection point came with GPT‑OSS; Gemma 4 pushed it over the line for multi‑step/looped workflows that were “impossible” locally ~6 months ago.
  - gemma-4-12b-qat stands out for speed/size with minimal accuracy loss; raises interesting architecture trade‑off questions for constrained hardware.

- How they run agentic flows:
  - Harness: Pi (as the agent) talking to a local inference server.
  - Server: LM Studio exposing an OpenAI‑compatible endpoint; they map Pi to the local model (e.g., google/gemma-4-12b-qat). They note plain llama.cpp would likely be faster.
  - Security: Every session runs inside Docker with tight permissions (bash‑only; no Python/web by default). They may enable curl in a separate image for controlled web access.

- Config highlights (condensed):
  - Point Pi to http://host.docker.internal:1234/v1 using the OpenAI‑completions API and a local model ID.
  - Docker Compose mounts Pi configs/sessions and sets OPENAI_API_BASE to the LM Studio endpoint; there’s a “sandbox” mode for even stricter isolation.
  - A simple wrapper script toggles sandboxing and starts the containerized agent.

- Takeaways:
  - Local models now cover a surprising swath of dev tasks, especially “personalized Google/docs lookups” and structured code refactors/tests.
  - Still not perfect: frontier models remain ahead on quality/recency, and memory demands can be heavy. But the offline, private, low‑latency workflow is finally practical on consumer hardware.

- Want to try it?
  - Install LM Studio (or llama.cpp), download Gemma 4 (12B QAT for speed or 26B A4B for quality), expose an OpenAI‑compatible endpoint, point an agent harness like Pi at it, and run inside Docker with least‑privilege.
