### Multi-Agentic Software Development Is a Distributed Systems Problem

Highlights

- Formalization: Given a prompt P, there are many consistent programs Φ(P). Spawning agents A1…An to implement parts φ1…φn only succeeds if there exists a single φ ∈ Φ(P) that each φi refines—i.e., they must agree on one interpretation.

- Rebuttals addressed: 
  - “Just specify the prompt”: Unless you write code-level specs, natural language remains ambiguous.
  - “It’s not really concurrent”: Once you use multiple agents working in parallel, it is.

Takeaway: Treat multi‑agent LLM systems like distributed protocols—design for agreement, interface contracts, and verification. AGI won’t save you from consensus.
