- Copilot CLI will be the primary tool; Microsoft is investing to deepen integration and is pushing engineers to file bugs/feedback.

HN: LLMs asked to “build the Pantheon” in OpenSCAD — who did it best?

What they did
- ModelRift benchmarked several AI coding tools on a single task: generate a parametric OpenSCAD model of Rome’s Pantheon from two reference images, iterating via the OpenSCAD CLI to render PNG previews.
- Chosen target: the Pantheon balances radial symmetry, columns, pediment, oculus, and clean solids—ideal for OpenSCAD’s booleans, extrusions, loops, and symmetry (vs. sculpting).
- Rationale: OpenSCAD’s text-first, parametric code maps well to how LLMs reason; changes are transparent and reproducible.

Key results (speed vs. quality)
- Best autonomous: Google Antigravity 2.0 + Gemini 3.5 Flash High — ~12 min, 4.5/5 quality. Used real Pantheon dimensions, included the front inscription, and uniquely modeled the interior coffered dome.
- Human-in-the-loop winner: ModelRift + Gemini Flash 3.0 — ~10 min, 3.8/5 quality using iterative annotations; strong but less detailed than Antigravity.
- OpenAI Codex: 3.0/5 quality with strong detail density (incl. inscription), but final STL didn’t match the PNG preview.
- Claude Code 2.1 + Sonnet: Slowest, 3.4/5; clean massing and proportions.
- Claude Code 2.1 + Opus: 3.0/5; clearer structure than Cursor but too monochrome.
- Cursor 3.5 + Composer 2.5: Fastest run, weakest output (1.4/5); off proportions and poor architectural details.

Why it matters
- LLMs can produce recognizable, parametric CAD directly in code (loops, arrays of columns, boolean oculus cuts), aligning with CAD/3D printing workflows (STL/3MF).
- Code-first CAD beats tool-control agents for this class of geometry, but challenges remain: long-range spatial planning, color/material discipline, and reliable exports.
- Human feedback meaningfully boosts results. Scores are benchmark-specific, not general model rankings.

Because Apple makes its money on devices, they are highly incentivized to "commoditize their complement" by pushing the development of highly capable, cheap, on-device local LLMs. If the tech giants succeed in making local models powerful enough for common tasks, it will act as a natural ceiling to the soaring API prices of closed AI labs.

### **The Bottom Line**
The era of unlimited, flat-rate access to GPT-4 and Claude might be ending, but the Hacker News community largely views this as a problem for the API providers, not the ecosystem. As frontier labs get bogged down in billion-dollar compute deficits, the open-weight and local-model ecosystems are waiting in the wings to offer developers cheap, sustainable, and highly efficient alternatives.
