Headline: That $5,000/month Claude Code “compute” claim is retail API spend, not real costs

- Forbes cited a claim that Anthropic’s $200/month Claude Code Max can consume ~$5,000 in compute. The post argues that number confuses retail API pricing with actual serving costs.
- Anthropic’s Opus 4.6 API list prices are ~$5 per million input tokens and ~$25 per million output. At those prices, a heavy Max user can indeed hit ~$5,000/month in “API-equivalent” usage.
- But OpenRouter pricing for comparable large MoE models suggests real inference costs are closer to 10% of Anthropic’s retail price:
  - Qwen 3.5 397B: ~$0.39/MTok input, ~$2.34/MTok output (vs $5/$25)
  - Kimi K2.5: ~$0.45/$2.25
  - Cache reads: ~$0.07/MTok (DeepInfra on Kimi) vs Anthropic’s $0.50
- If so, a power user “burning” $5,000 at Anthropic’s API rates likely costs Anthropic around $500 in compute—not $5,000. The worst-case loss per true edge-case user is closer to a few hundred dollars, not thousands.
- Most subscribers don’t hit those extremes: Anthropic says <5% reach weekly caps; its /cost data pegs average usage at ~$6/day API-equivalent (~$180/month), implying roughly ~$18/month in actual serving cost at a 10% ratio—break-even or profitable against $20–$200 subscriptions.
- The $5,000 figure likely is real for Cursor specifically, because it pays Anthropic (near) retail to offer Opus 4.6—leaving wrappers/IDEs squeezed when users insist on Anthropic models for quality/brand reasons.
- Bigger picture: inference probably isn’t the main money sink; training runs, top-tier talent, and multi‑billion dollar compute commitments dominate. The “inference is a money pit” narrative props up 10x+ API markups and deters competition.
- Takeaway: don’t conflate API list prices with serving costs. Competitive OpenRouter rates are a better proxy for real inference economics—and they’re a fraction of frontier labs’ retail pricing. Caveat: exact model comparability and provider margins vary, but the order-of-magnitude gap is hard to ignore.

### Is legal the same as legitimate: AI reimplementation and the erosion of copyleft

#### [Submission URL](https://writings.hongminhee.org/2026/03/legal-vs-legitimate/) | 546 points | by [dahlia](https://news.ycombinator.com/user?id=dahlia) | [543 comments](https://news.ycombinator.com/item?id=47310160)

AI reimplementation of a core Python library sparks a copyleft legitimacy fight

- What happened: Dan Blanchard, maintainer of chardet (a hugely used text-encoding detector), shipped v7.0: a ground-up rewrite listed with Anthropic’s Claude as a contributor. It’s 48x faster, multicore, and—crucially—relicensed from LGPL to MIT. Blanchard says he never read the old code, fed only the API and tests to Claude, and measured <1.3% similarity via JPlag—so he treats it as an independent work not bound by the old copyleft.

- The pushback: Original author Mark Pilgrim objects that a maintainer long exposed to the code can’t credibly claim “clean room,” and that LGPL’d work shouldn’t be laundered into permissive licensing. Armin Ronacher (Flask) welcomes the change; Salvatore Sanfilippo (antirez, Redis) defends AI reimplementation as lawful under copyright’s idea/expression split.

- The essay’s argument: Legality isn’t the question—legitimacy is. GNU’s historical clean-room efforts turned proprietary software into free software, expanding the commons. Here the vector runs the other way: from copyleft to permissive, removing reciprocity that kept improvements in the commons. The GPL doesn’t stifle sharing; it conditions distribution on sharing back. MIT enables one-way extraction by better-resourced actors. If AI makes “from-scratch” rewrites cheap, copyleft protections can be sidestepped in practice, eroding norms even if the law allows it.

- Why it matters: This sets a precedent for using LLMs to “wash” licenses by reimplementation. It raises governance questions for popular projects, the durability of copyleft in an AI era, and whether open-source stewardship should prioritize what’s merely legal over what sustains the commons.

Based on the discussion, here is a summary of the comments:

**License Laundering vs. Clean Room Engineering**
The community is debating whether using an LLM constitutes a valid "clean room" implementation or a mechanism to bypass copyleft obligations (often termed "license laundering"). While some argue that if the prompt was just the API surface and tests, the resulting code is a new expression of the same ideas (violating no copyright), others contend that LLMs are statistically rewriting their training data—which likely included the original GPL code—making the output a derivative work that retains the original license.

**The "Tests as Source" Argument**
A significant thread focuses on the specific methodology: feeding the original test suite to the AI to generate passing code.
*   Commenters argue that if the test suite is comprehensive enough to dictate the implementation, the tests function like a "machine-readable contract" or source specification.
*   One user suggests the AI is effectively acting as a "compiler" for the tests; if the tests are LGPL, the "compiled" output might arguably be derivative, similar to how decompiling a binary doesn't free it from copyright.
*   Comparisons are made to *Oracle v. Google*, questioning where the line between functional APIs (non-copyrightable) and creative expression lies when an AI does the writing.

**Copyright Status of AI Output**
There is confusion and disagreement regarding the copyright status of the generated code itself.
*   Some users point out that the US Copyright Office has thus far refused to register AI-generated works, suggesting the output might be public domain (which Blanchard could arguably wrap in an MIT license, though he couldn't claim authorship).
*   Others argue that if the output is indistinguishable from human code, it creates a "Schrodinger’s Copyright" situation where legality depends entirely on the unprovable assertion of whether a human or machine wrote it.

**Provenance and "Pierre Menard"**
The discussion touches on the philosophical dilemma of identical recreation, with one user referencing *Pierre Menard, Author of the Quixote* (a story about a man who re-writes *Don Quixote* word-for-word independently). Users worry that without clear legal definitions, bad actors will claim human-written code was AI-generated to strip licenses, or claim AI code was human-written to gain copyright protections.
