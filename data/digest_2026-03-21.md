Tiny Corp pitches tinygrad—and ships “tinybox” DL rigs, with an audacious exaflop teaser

- The framework: tinygrad centers on three OpTypes—ElementwiseOps (unary/binary/ternary like SQRT/ADD/WHERE), ReduceOps (SUM/MAX), and MovementOps (RESHAPE/PERMUTE/EXPAND) that are copy-free via ShapeTracker. CONV/MATMUL are left as “read the code to solve this mystery.”
- Why it might be fast: per-op custom kernel compilation for shape specialization, lazy tensors for aggressive fusion, and a much simpler backend. Today it’s often not faster than PyTorch, but the goal is to beat PyTorch 2x on a single NVIDIA GPU by next year (and be good on Apple M1).
- Status/use: Full autodiff (forward/backward), ONNX loading, and training supported. It powers openpilot’s driving model on Snapdragon 845 GPU, replacing SNPE. Still in alpha; “fairly stable” but caveat emptor.
- Hiring and bounties: Tiny Corp is funded and hiring (software, plus ops/hardware). Contributing to tinygrad strongly influences consideration. Bounties pay while gauging fit. Contract work/sponsorships welcomed.
- Hardware: tinybox deep-learning systems are “now shipping”
  - Red v2: 4x 9070XT, 64 GB total GPU RAM, 32-core AMD EPYC; $12,000
  - Green v2: 4x RTX Pro 6000 (Blackwell labeled), 384 GB GPU RAM, 32-core AMD Genoa; $65,000
  - Shipping within a week after wire payment; pickup in San Diego + worldwide shipping. No customization, wire transfer only; W‑9 available.
  - Teasers: a “blackwell” line note and an “exabox” (2027, ~$10M) claiming ~1 exaflop FP16, 720 GPUs, 600 kW, 20x8x8.5 ft footprint—clearly moonshot territory.
- Vibe and mission: irreverent tone (“Invest with your PRs”), aggressive cost/perf claims (MLPerf Training 4.0 vs machines 10x the price), and a stated goal to “commoditize the petaflop and enable AI for everyone.”

The takeaway: pursue AI disputes in court without cutting off libraries that preserve history.

Atuin v18.13: big upgrade for shell history, fuzzy search, and an opt‑in AI

What’s new
- Much faster fuzzy search via a revamped daemon: keeps a hot in‑memory index (powered by a modified nucleo implementing fzf’s algorithm) with tunable frequency/recency/frecency multipliers. Enable via atuin setup or set search_mode="daemon-fuzzy" and daemon autostart.
- Atuin AI (opt‑in): an English‑to‑bash helper inside your shell. Press ? on an empty prompt to describe a task; Enter to run, Tab to edit, f to follow up. Uses frontier LLMs plus a dataset from man pages/command outputs.
  - Safety: suspects dangerous commands and requires a double Enter; mix of static checks and LLM guardrails.
  - Privacy: by default only knows OS and shell; can request optional, permissioned context (cwd, contents, git status) later.
- Hex pty proxy: a tiny, tmux‑like proxy that lets Atuin render popups over existing output without clearing the screen or going fullscreen, then cleanly restore output. Enable with eval "$(atuin hex init)". Designed to be minimal, not impact scrollback, and just proxy bytes.
  - Future: search/sync command outputs, support shells without hooks, broader readline‑style integration.
- Hosted sync auth refresh (Atuin Hub): login with Google/GitHub and email recovery. Existing creds still work. Hub isn’t open source yet (planned once it stabilizes). Self‑hosters unchanged.

Why it matters
- Speed and relevance: in‑memory fuzzy index plus scoring multipliers should make shell history search feel instant and smarter.
- Better UX: Hex fixes the old fullscreen/clear trade‑off without the overhead of tmux.
- Practical AI in the terminal: quick command generation with explicit safety and privacy boundaries.

Notable fixes and polish
- Big daemon memory reduction, better Nushell compatibility, improved multiline handling in fish, more robust DB behavior, proper Ctrl‑C in AI, and smoother setup.

How it works
- Root and phase instruction files (CLAUDE.md and CLAUDE.<phase>.md) guide agents’ behavior per phase.

- Phase gates enforce preconditions before moving forward; per‑component directories are created during decomposition.
- Manual edits are discouraged; all changes flow through an agent to ensure procedures and decision capture.

Getting started
- Recommended: npx degit pangon/ai-sdlc-scaffold my-project, remove repo-specific files, git init, then run /SDLC-init in Claude Code.

Caveats
- Geared to Claude Code; other agent setups may need adaptation.
- Process‑heavy by design; overhead might feel rigid for very small or exploratory projects.

Bottom line
- A thoughtful “AI-first SDLC” blueprint that many HN readers experimenting with agentic development and repo‑centric workflows may find useful.
