#### [Submission URL](https://github.com/califio/publications/blob/main/MADBugs/CVE-2026-4747/write-up.md) | 264 points | by [ishqdehlvi](https://news.ycombinator.com/user?id=ishqdehlvi) | [102 comments](https://news.ycombinator.com/item?id=47597119)

CVE-2026-4747: Remote kernel RCE in FreeBSD’s NFS RPCSEC_GSS (kgssapi.ko)

What happened
- A stack buffer overflow in FreeBSD’s RPCSEC_GSS server-side validation lets an attacker achieve remote kernel code execution via NFS, yielding uid 0.
- Affected: FreeBSD 13.5 (<p11), 14.3 (<p10), 14.4 (<p1), 15.0 (<p5). Tested on 14.4-RELEASE amd64 (GENERIC, no KASLR).
- Advisory: FreeBSD-SA-26:08.rpcsec_gss.

Root cause
- In svc_rpc_gss_validate(), the code reconstructs an RPC header into a 128-byte on-stack buffer, then memcpy’s the RPCSEC_GSS credential body without checking its length.
- Only 96 bytes are actually available after fixed header fields; larger credentials overflow into saved registers and the return address.
- The fix adds a bounds check that rejects credentials exceeding the remaining buffer space.

Reachability and constraints
- Trigger path: RPCSEC_GSS auth (flavor 6) with GSS DATA on NFS (port 2049/TCP), after a valid context lookup and replay check.
- Crucially, the attacker must possess a valid Kerberos ticket for the NFS service principal; without it the code bails out earlier and the memcpy isn’t hit.
- XDR caps credential size at 400 bytes, leaving enough headroom for a reliable overflow.
- The write-up demonstrates a working remote exploit that pivots to a root shell.

Why it matters
- This is a remotely reachable kernel-level RCE in a widely deployed network service, especially in enterprise environments using NFS with Kerberos.
- Even low-privilege users with valid Kerberos tickets can compromise the server.

What to do
- Patch immediately: 13.5-p11, 14.3-p10, 14.4-p1, 15.0-p5 (or newer).
- Interim mitigations if you can’t patch fast:
  - Disable NFS Kerberos (RPCSEC_GSS) or unload kgssapi.ko where possible.
  - Restrict/segment NFS (2049/TCP) to trusted networks; firewall untrusted sources.
  - Monitor for anomalous NFS auth activity and kernel crashes/panics.

- Big picture: Once the HJB/HJ link is visible, it lines up continuous‑time RL with physics and opens a control‑theoretic lens on modern generative modeling—interpreting diffusion model training as a stochastic optimal control problem, adjacent to optimal transport.

Why it matters
- Unifies RL, generative modeling, and classical mechanics under a common PDE/control framework.
- Offers principled routes to algorithms (via value functions/viscosity theory) and deeper interpretations of diffusion training objectives.
