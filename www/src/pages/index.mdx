- The entire 209GB model is streamed on demand from SSD via a custom Metal pipeline—no frameworks, no Python.

Why it matters
- Shows that gigantic MoE models can run locally on commodity laptops by activating only a handful of experts per token and streaming weights from fast NVMe, relying on the OS page cache.
- Demonstrates how careful kernel design and Apple’s unified memory model can hit high throughput without custom caching or heavyweight stacks.

Key results
- 4-bit experts (FMA-optimized): 4.36 tok/s, “Excellent” quality, tool calling works, 209GB on disk.
- 4-bit baseline (pre-FMA): 3.90 tok/s.

- Measured SSD read: 17.5 GB/s; GPU dequant kernels saturate ~418 GiB/s memory bandwidth.

How it works
- Architecture: 60 layers total—45 GatedDeltaNet (linear attention) + 15 full attention. Each layer has 512 experts; only K=4 (plus a shared expert) run per token. Hidden dim 4096.
- SSD expert streaming: parallel pread() loads only the active experts (~6.75MB each) per layer; “Trust the OS” for caching.
- FMA-optimized dequant: reformulates (nibble*scale + bias)*x into fma(nibble, scale*x, bias*x), letting the GPU do dequant+multiply in one fused instruction (~12% speedup).
- Metal compute shaders: custom kernels for 2/4‑bit dequantized matvec, fused SwiGLU, RMSNorm, batched attention (QK^T/softmax/V), RoPE, MoE combine/residual/sigmoid gate.
- Deferred expert compute: submit expert pass without waiting; overlaps CPU prep for the next layer.
- Accelerate BLAS speeds GatedDeltaNet state updates (cblas_sscal/sgemv/sger), ~64% faster than scalar.
- Pipeline is intentionally serial (GPU → SSD → GPU): on Apple Silicon, SSD DMA and GPU compute share a memory controller; overlapping causes latency spikes.

Caveats
- 2-bit mode is faster but breaks structured outputs and tool calling; 4-bit is the “production” path.
- Requires fast NVMe and Apple Metal; heavily optimized for M3 Max unified memory.
