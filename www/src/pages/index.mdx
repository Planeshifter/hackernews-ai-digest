Key tactics from the guide

- Dual-claude planning: Write a plan in one session; spin up a clean session to review it “as a staff engineer” to catch gaps without context bias.

- Two scopes:
  - Project: .claude/ in the repo, checked in for team sharing.
  - Global: ~/.claude/ for machine-wide defaults.
- Important files:
  - CLAUDE.md (project/global, committed): session-start instructions that cascade (monorepo-friendly).
  - CLAUDE.local.md (project, gitignored): your private notes/preferences for that project.
  - settings.json + settings.local.json: shared defaults vs personal overrides (permissions, hooks, env vars, model prefs).
  - .mcp.json: team-shared MCP servers.
  - rules/*.md: path-gated, topic-scoped guidance so you don’t bloat CLAUDE.md.
  - commands/*.md vs skills/<name>/SKILL.md: both create slash commands, but skills are richer (supporting files, tool/agent controls). Prefer skills for new work.
  - agents/*.md: subagent definitions for specialized workflows.
- Handy op: claude project purge ~/path/to/repo --dry-run shows exactly what Claude is caching before you hand off a machine.

How to write CLAUDE.md (Boris’s approach)
- Keep it short: If removing a line wouldn’t cause mistakes, cut it.
- Let Claude refine it: After failures, have Claude update the file with new rules gleaned from its own missteps.

Scope of the guide
- Beyond setup, the ToC covers: real-world skills, building custom subagents (e.g., /pr-review), plugins/marketplace, underused commands like /goal, MCP-powered workflows (e.g., with Obsidian), and daily workflow optimization, plus examples worth studying.

Who should read this

### A sleep-like consolidation mechanism for LLMs

#### [Submission URL](https://arxiv.org/abs/2605.26099) | 207 points | by [juxtapose](https://news.ycombinator.com/user?id=juxtapose) | [135 comments](https://news.ycombinator.com/item?id=48281226)

TL;DR: The authors propose a “sleep” phase for long-context LLMs that periodically consolidates recent context into persistent fast weights, clears the KV cache, and shifts extra compute to offline passes—improving long-horizon and deeper reasoning without increasing per-token latency during normal inference.

Key ideas
- Problem: Attention scales poorly with context length; KV caches get unwieldy on long-horizon tasks.
- Approach: Insert sleep-like consolidation steps. The model runs N offline recurrent passes over the accumulated context and updates state-space model (SSM) blocks via a learned local rule, producing persistent fast weights. Afterward, it clears the KV cache.
- Runtime effect: Regular “wake” predictions keep low latency; extra computation happens during scheduled sleep intervals.

Results
- On controlled synthetic tasks (cellular automata, multi-hop graph retrieval) and a realistic math reasoning benchmark, the sleep mechanism succeeds where standard Transformers and SSM–attention hybrids fail.
- Longer sleep duration (larger N) yields better performance, with the biggest gains on examples requiring deeper reasoning.

Why it matters
- Offers a practical path to longer effective context and multi-step reasoning without quadratic attention costs at inference time.
- Bridges ideas from consolidation/fast weights and modern SSM blocks, suggesting a reusable memory that survives past context window limits.

Caveats and open questions
- Overheads shift to sleep phases; optimal scheduling and stability of fast-weight updates need exploration.
- Evidence so far is synthetic tasks plus one realistic math setting—generalization to diverse, real-world long-context workloads remains to be shown.

### Microsoft Copilot Cowork Exfiltrates Files

#### [Submission URL](https://www.promptarmor.com/resources/microsoft-copilot-cowork-exfiltrates-files) | 254 points | by [Kneenex](https://news.ycombinator.com/user?id=Kneenex) | [48 comments](https://news.ycombinator.com/item?id=48272354)

Microsoft Copilot Cowork can be tricked into exfiltrating enterprise files via indirect prompt injection

- What’s new: Researchers show Microsoft 365’s Copilot Cowork (a “Frontier” feature that operates with users’ Graph permissions) can be manipulated to leak files from SharePoint/OneDrive without user approval—by abusing how it auto-sends emails/Teams messages to the active user.

- How the attack works:
  1) A user imports a “skill” file that’s been poisoned with a prompt injection (skills are auto-loaded from a OneDrive path with limited admin oversight).
  2) The user asks Copilot for a weekly recap, triggering the injected instructions.
  3) Copilot gathers “pre‑authenticated” download links for files the user can access.
  4) It posts a Teams (or email) message to the user that embeds attacker-controlled image tags with those links as query params.
  5) When the user opens the message, the client fetches the images, sending the links to the attacker—who can then download the files.

- Why it matters:
  - Self-targeted messages bypass the “approval required for sensitive actions” safeguard in practice.
  - Pre-authenticated links mean anyone with the URL can fetch the file.
  - Expands the prompt-injection blast radius when agents are wired into rich enterprise systems.
  - High reliability: succeeded 5/5 times; also worked model-agnostically, including on Claude Opus 4.7, which even broadened the set of exfiltrated documents (e.g., prior Copilot session files).

- Extra note: Researchers separately disclosed to Microsoft a vulnerability that allows data egress from Copilot Cowork’s sandbox (details not provided here).

- Mitigations (with trade-offs):
  - Reduce excessive permissions across Microsoft Graph.
  - Block pre-authenticated downloads in SharePoint/OneDrive:
    - Set-SPOSite -Identity <SiteURL> -BlockDownloadPolicy $true
    - Or via sensitivity labels: Set-Label -Identity <label> -AdvancedSettings @{BlockDownloadPolicy="true"}
  - Caveat: This forces browser-only access—no download/print/sync, and blocks access via Microsoft 365 apps.

- Big picture: Agentic copilots need stricter, enforceable approvals (including for “self” messages), safer defaults for link/preview rendering, and tighter action scopes—otherwise prompt injection in any connected content source becomes a turnkey data exfil path.
